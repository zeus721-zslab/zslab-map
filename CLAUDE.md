# 작업 규칙 (공통)

## 작업 방식
1. 작업 시작 전 PROGRESS.md에 STEP 항목 먼저 기록
2. 각 STEP 완료 직후 즉시 [x] 업데이트 (일괄 금지)
3. 에러 시 PROGRESS.md에 `❌ 에러: <내용>` 기록 후 중단·보고
4. 묻지 말고 스스로 판단 진행

## PROGRESS.md 스텝 형식
## STEP N: 작업명 (날짜)
- [ ] STEP N-1: 세부 작업
- [ ] STEP N-2: 세부 작업
- [x] STEP N-3: 완료된 작업

파일 위치: ~/PROGRESS.md

## 코딩 원칙

### 1. 코딩 전에 생각하기
- 가정이 있으면 명시. 불확실하면 묻기.
- 해석이 여러 개면 제시 — 조용히 선택하지 말 것.
- 더 단순한 방법이 있으면 말할 것. 필요하면 반박.

### 2. 단순함 우선
- 요청하지 않은 기능 추가 금지.
- 단일 사용 코드에 추상화 금지.
- 200줄로 쓰고 50줄로 가능하면 다시 쓸 것.

### 3. 최소 변경 원칙
- 인접 코드·주석·포맷 개선 금지.
- 고장나지 않은 것 리팩토링 금지.
- 기존 스타일 유지.
- 관련 없는 dead code 발견 시 삭제 말고 언급만.
- 변경된 모든 줄은 사용자 요청으로 직접 추적 가능해야 함.

### 4. 목표 기반 실행
멀티스텝 작업은 계획 먼저:
[작업] → 검증: [확인 방법]

## 토큰 절약
- 프롬프트는 핵심만 간결하게.
- 불필요한 설명·반복·과도한 검증 시나리오 금지.

## 프로젝트 규칙
PROJECT.md가 존재하면 작업 시작 전 반드시 읽어라.
